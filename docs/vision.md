
# Vision


- Build a distributed key-value store.
- Support multiple data types.
- Support multiple data formats.
- Support multiple storage engines.
- Support some capabilities of modern storage engines as close as possible.



## How to achieve
- Incremental updates to the existing architecture and refactoring it to support the new features.


## Motivation

- To provide a flexible and scalable solution for storing and retrieving data.
- To support a wide range of use cases and applications.
- To enable efficient and reliable data management.


## Purpose

- To build and learn how modern storage engines really work under the hood.
- To implement the learnings from modern storage engines in a practical way.
